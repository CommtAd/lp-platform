# CLAUDE.md — LP制作規約（営業向け）

このリポジトリで新しいLPを制作・修正するときは、以下の規約に必ず従うこと。
基盤の責務（公開・タグ注入・CV受信と転送）を壊さないための最小限のルール。

## 基本ルール（lp-platform-spec.md §9）

1. LPは `apps/lp/clients/{slug}/` 配下に作成する。
   ベーステンプレート（`clients/_base-a/`）をコピーして開始する。
2. **LPShell で全体をラップすること（削除・迂回禁止）。**
   タグ注入・noindex・UTM保持はすべて LPShell が担う。外すと機能が失われる。
3. **フォームは LPForm を使用する。送信先URLの変更禁止。**
   送信先は基盤側で固定（`lib/form-endpoint.ts`）。HTML納品時も動作させるため。
   フィールド構成・見た目は `config.ts` の `form` で自由にカスタマイズしてよい。
4. `tel:` リンク・LINEリンクには `trackEvent('tel_tap' | 'line_tap', slug)` を付与する。
5. それ以外のデザイン・構成・アニメーションは自由。
6. 画像は `clients/{slug}/` 配下に置き、`config.ts` からパス参照する
   （`public/clients/{slug}/` に配置し、`ImageSlot` で aspect-ratio + object-fit: cover 受け）。
7. 公開フロー: PR作成 → プレビューURLで顧客確認 → main マージ →
   ダッシュボードで公開ボタン（status を published に切り替え）。

## 制作フローの追加ルール（implementation-brief-01.md §6）

8. **新規LP制作時は必ず該当 brief.md を読み、`design-refs/`（参照画像）を確認してから着手する。**
9. **Claude Design の出力コードを直接持ち込まない。**
   ベーステンプレート（`_base-*`）からの差分適用で制作する。
10. **コード化後のデザイン修正はコード側で行う。Claude Design には戻らない（一方通行ルール）。**
11. **参照画像のデザインは参考にしてよいが、画像素材そのものをLPに流用しない。**
    参照画像は見た目の検収基準。実際の画像は顧客素材またはプレースホルダを使う。

## テンプレA（`_base-a`）と実顧客LPの関係

- `_base-a/` = パターンAの原型テンプレ。テキストはダミー、画像はプレースホルダ。
  **テンプレA自体の改善はこのフォルダで行う。**
- `the-personal-pilates/` = 実顧客LP。実データで運用する本番。
- `page.tsx` / `FaqList.tsx` は両者で同一のコピー。差分は `config.ts` のみ。
  新規顧客は `_base-a/` をコピーし、`config.ts` を実データに差し替える。

## 保持すべき確定デザイン要素（パターンA・変更禁止）

- 配色: ネイビー × ゴールド × 生成り。見出しは明朝、本文はゴシック。
- セクション構成（順序固定）:
  ヘッダー（オファーバー）→ FV（縦書きキャッチ）→ 90分お試し体験（0円 + 6項目の円形アイコン）
  → 入会金0円 → 料金 → スタジオについて → 悩み共感4枚 → 選ばれる3つの理由
  → 通い方シーン3例 → トレーナー紹介（スワイプ）→ 90分体験の流れ3ステップ
  → FAQ4問 → 店舗案内2店 → 予約フォーム。
- 「業界最安クラス」は削除済み。復活させない。
- 二重価格表記（11,000円→0円 / 33,000円→0円）は顧客確認済み。そのまま維持。
